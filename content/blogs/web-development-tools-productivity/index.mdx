Networking is a common term that anyone who uses computers (phones, printers, laptops,
desktops, iPads, etc.) has likely heard at least once in their life.

However, not everyone clearly and deeply understands what a network actually is — even
developers who have spent 3–4 years at university 🙂

For those working as System Administrators or in DevOps, networking is one of the most
critical areas that requires a deep understanding 😢

This blog is the first article in a Linux Networking series. By following this journey, you will
gain a solid and in-depth understanding of Linux networking.

## What is a network?

In everyday life, a network is a group of people connected by at least one common
relationship, allowing them to communicate with each other.

You can use this idea as an analogy for computer networking.

In computing, a network is a collection of devices (called nodes, such as computers, phones,
or servers) that are connected in a way that allows them to exchange data.

The "connection" between these nodes can be physical (like cables) or wireless (like Wi-Fi),
but the goal is the same: enable communication between devices.

## Purpose?

Going back to the earlier analogy:

If people are not connected to others, they are limited to what they already know. They only
interact with their immediate environment and don't gain new information from the outside
world.

Computers work in a similar way.

Without a network, a computer can only access:

- Its own hardware
- The data stored on its local disk

It is isolated.

A network removes that limitation.

With networking, computers can:

- Access data from other machines
- Communicate with services (APIs, databases, servers)
- Share resources without physically moving data

In other words, the network acts as a bridge for data exchange.

Instead of copying files manually (e.g., using USB drives or moving storage devices),
systems can send and receive data instantly over the network.

## Components?

What are the components required to create a network?

- **Endpoints (Hosts):** Devices that generate or consume data (e.g., servers, clients)
- **Network Interfaces:** Hardware or virtual interfaces that connect a device to a network
- **Transmission Media:** Physical (Ethernet cables, fiber) or wireless channels
- **Networking Devices:** Routers, switches, and access points that direct traffic
- **Protocols:** Defined rules for communication (e.g., IP, TCP, UDP, HTTP)

Each component plays a role in ensuring that data reaches its intended destination correctly.
We will dive deeper into each component in later posts.

## Types of Network

| Type of Network | Definition | Scope / Size | Example | Key Characteristics |
| --- | --- | --- | --- | --- |
| Local Network (LAN) | A network of devices connected within a small, limited area | Small (home, office, room) | Home Wi-Fi connecting laptop, phone, TV, printer | Devices are physically close; uses routers/switches; fast and low latency; easy to set up |
| Private Network | Multiple local networks connected together and restricted to specific users | Medium to large (company, school, organization) | Company network with separate departments (Engineering, HR) | Access is controlled (not public); can segment networks; more secure and isolated; may partially connect to the internet |
| Public Network | A network that anyone can access and use | Very large (global scale) | The Internet | Open to everyone; maintained by ISPs; connects millions of networks; requires regulation and infrastructure |

## Network in Linux

> "For the top 500 most powerful supercomputers, Linux distributions have held 100% market share since 2017."

Linux dominates in infrastructure, cloud environments, and high-performance systems.

**Why Linux?**

You should take time to investigate that question yourself. The answer is not a single feature,
but a combination of factors: control over the kernel, strong networking capabilities, stability,
and flexibility at scale.

Understanding why Linux is dominant will give you deeper insight into how modern systems
are designed.

For now, this article focuses specifically on networking in Linux — how to inspect and
control it in practice.

### List of network hardware?

In Linux, you can inspect the physical network devices (e.g., Wi-Fi cards, Ethernet adapters)
using:

```bash
sudo lshw -class network
```

This command provides detailed information about each network device, including:

- Hardware type
- Driver in use
- Device status

If the command is not available, `lshw` is likely not installed. Install it with:

```bash
sudo apt install lshw
```

### List of network interfaces

A network interface is the point where your system connects to a network. It can be:

- **Physical** (Ethernet, Wi-Fi)
- **Virtual** (used by containers, bridges, tunnels)

Most modern systems include built-in interfaces such as Wi-Fi and Ethernet.

To list all network interfaces on your system:

```bash
ip addr show
```

This command displays:

- Interface names (e.g., `eth0`, `wlan0`, `enp0s3`)
- IP addresses
- Interface states (`UP`/`DOWN`)

Returning to the network components: a network interface is a hardware or virtual interface
that connects a device to a network.

- **Hardware interfaces:** physical components such as Ethernet ports or Wi-Fi adapters
- **Virtual interfaces:** software-defined interfaces used by bridges, containers, or tunnels

### Turn on / Turn off a network interface

Linux allows direct control over the state of network interfaces.

**Bring an interface up:**

```bash
sudo ip link set dev <INTERFACE_NAME> up
```

**Bring an interface down:**

```bash
sudo ip link set dev <INTERFACE_NAME> down
```

**Example:**

```bash
sudo ip link set dev eth0 down
sudo ip link set dev eth0 up
```

> If an interface is down, the system cannot send or receive data through it, regardless of other configurations.
